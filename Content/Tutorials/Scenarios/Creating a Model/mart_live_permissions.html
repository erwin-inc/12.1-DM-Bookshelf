Mart Live Permissions

In the earlier versions of erwin DM, the permissions on a Mart model were verified only when you saved the model. Now, your permissions are verified on every action that you perform. Rules govern these permissions.

The following rules apply when you work on Mart models:

  • Rule 1: Permissions are taken for granted for objects that Mart does not manage. For example, Mart does not manage drawing objects and annotations.
  • Rule 2: Permissions apply to only those objects that are created when you are connected to Mart. For example, you added five tables to your model before saving it as a Mart model. Suppose that you do not have the permissions to modify tables on Mart. You can still modify the five tables that you created before saving the model to Mart.
  • Rule 3: Permissions do not apply to local models.
  • Rule 4: The permissions on your offline models are same as your Mart models. For example, you are working on a Mart model and you do not have permissions to add views to a model. If you save your model as an offline model, you cannot add views even to your offline model.
  • Rule 5: Offline models inherit the current permissions of the user, when the models are bound to Mart again. For example, you do not have permissions to add views to a Mart model. You save a Mart model as an offline model and update it. If your permissions on Mart have changed and you can add views, you can add views after binding the model back to Mart.
  • Rule 6: After an offline model is bound to Mart and the model inherits the permissions of the current user, Mart does not verify permissions again.
