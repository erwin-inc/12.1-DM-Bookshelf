Change Management in erwin Data Modeler Workgroup Edition

Change management in erwin� Data Modeler Workgroup Edition is based on a check-out/check-in paradigm. You open a model from the mart into a desktop client and then you save the model back into the mart. The master model in the mart is updated with your changes, and a model version of the model as it existed before your changes is created.

In a workgroup modeling environment, you and other users can be working on the same model at the same time. For example, if you opened a model from the mart and you do not lock it, another modeler can save changes to the model.

When you save your changes, if another user made changes to the same model, a Change Management dialog lets you compare your changes with changes made by other modelers in the workgroup environment.
