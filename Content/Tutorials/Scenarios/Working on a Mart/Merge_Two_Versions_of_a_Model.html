Merge Two Versions of a Model

Workgroup members can save a copy of a workgroup model locally, change it offline, and then merge their changes back into the master model version using the Complete Compare feature (erwin� Data Modeler only). If two modelers were working on the same model without locking it, you may want to merge the two versions of the model back to the mart.

To merge two versions of a model

  1. In the Version Manager dialog, use the control key to select each version of the model you want to compare and click DIFF.

    The Right Model Selection dialog appears.

  2. Click Compare.

    The Resolve Difference dialog appears.

  3. Compare the changes in the left and right change lists, and decide which changes you want to save and cancel. By default, erwin� Data Modeler cancels all of your conflicting changes and saves all of your non-conflicting changes to the master version of the model. You can take the following actions to resolve the differences:
    ToLeft

    Move the object or property in the compare row from the right model to the left model.

    ToRight

    Move the object or property in the compare row from the left model to the right model.

    Match

    Match two items that were not automatically matched. When you click this button the cursor changes to a double-ended arrow. Click on an item in the left model, then click on an item in the right model. The compare tree adjusts so that the items appear matched.

    Unmatch

    Break the match between an item in the left model and an item in the right model. Select the row that contains the items that are incorrectly matched, then click this button to move the selected items to different lines.

    Click Finish.

    The results are merged.

  4. Click Close.

    The Right Model Selection dialog closes.
