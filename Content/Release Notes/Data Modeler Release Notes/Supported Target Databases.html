Supported Target Databases

The application supports or is compatible with the following target databases:

  • ArangoDB (3.x)
  • Avro (1.x)
  • Amazon Keyspaces
  • Azure Synapse
  • Cassandra (3.x/4.x)
  • Couchbase (6.x, 7.x)
  • Databricks
  • Db2 for i (5.x/6.x/7.x)
  • Db2 for LUW (9.5, 9.7/10.x, 11.1)
  • Db2 for z/OS (8.1, 9.1/10/11, 12)
  • DynamoDB
  • Google BigQuery
  • Hive (2.1.x)

    Refer to the Limitations in Hive topic.

  • Informix (10.x/11.x/12.x)
  • JSON (1.x)
  • MariaDB (10.4.x)
  • MongoDB (4.x)
  • MySql (8.x)
    Use MySQL certified version 8.0.23 for Amazon Aurora.
  • Neo4j (4.2.x, 4.3.x)
  • Netezza (7.2)
  • ODBC/Generic (2.0, 3.0)
  • Oracle (12c r2, 18c, 19c, 21c)
  • Parquet (2.x)
  • PostgreSQL (9.6.x/10.x/11.x)
    Versions 9.6.21, 9.6.24, 10.16, 10.20, 11.11, 11.14, 12.6, 12.9, 13.2, and 13.5 are certified. You can also use PostgreSQL certified version 11.8 for Amazon RDS and Amazon Aurora.

    While reverse engineering from database using any version of PostgreSQL above 11.x, some objects and properties are not retrieved in the reverse engineered model.

  • Progress (9.x/10.x/11.x)
  • Redshift (1.0)
  • SAP ASE (15.x/16)
  • SAP IQ (15.x/16)
  • SAS (stabilized support)
  • Snowflake
  • SQL Server (2012, 2014, 2016/2017, 2019, Azure)
  • Teradata Database (14.10, 15.x/16.x)

The erwin Data Modeler 64-bit version is not verified for the Progress 9.x/10.x database.
