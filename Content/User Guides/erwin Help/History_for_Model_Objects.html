Configure History Tracking for Model Objects

You can save information about modeling events throughout the history of a model. You can use this information to gather information about the model, such as when the model was created, and when specific objects were changed. Use the features in the History Options tab of the Model Editor to set options for how historical information is tracked. You can view history events on the History tab of an object for which you are tracking history.

Follow these steps:

  1. Click Model, Model Properties to open the Model Editor.
  2. Go to the History Options tab and select from the following options:
    History Objects Save history for

    Specifies that you want to save history information for the selected objects.

    History Events Save history when an object is

    Specifies that history is tracked whenever the selected events occur for the objects that are selected in the Save history for list. For example, you can trace history when an entity is created, linked to a model source, migrated as a foreign key, reverse engineered, copied.

  3. Click Close to save your selections and close the Model Editor.

If you decide not to maintain the history and clear the "Save History When an Object Is..." check box for an object, the recording of history is suspended. If you select the check box again, gathering history information resumes starting from that point.
