Specify a Column Definition Using a Physical Domain

You can work with the features in the Comment tab of the Domain Editor in a physical model to specify a column definition using a physical domain.

To specify a column definition using a physical model

  1. Click Domains on the Model menu.

    The Domain Editor opens.

  2. Click the name of the domain in the Navigation Grid for which you want to specify a column definition.

    To filter a large list of domains, enter the name (or the first few characters) of the domain for which you want to specify a domain parent in the Enter filter text box.

  3. Click the Comment tab and work with the following options:
    Comment

    Lets you enter or edit a non-inheritable comment for the selected domain. Use the toolbar to cut, copy, paste, reset properties to restore inheritance, and edit or check the spelling of text.

    Definition Inherited by Attributes

    Lets you enter or edit the definition that the column inherits when it is associated with the selected domain. If you do not want to override existing column comments, use the %AttDef macro.

  4. Click Close.

    The column definition is specified and the Domain Editor closes.
