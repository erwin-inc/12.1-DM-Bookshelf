Define a Db2 LUW Work Class

Use the Db2 for LUW Work Class Editor to create and work with Db2 LUW work classes. You then must define a work action set, and associate it with a work class set for it to become active.

To define a Db2 LUW work class

  1. Right-click a Work Class Set in the Model Explorer and select Properties.

    The Db2 for LUW Work Class Set Editor opens.

  2. Click the Work Classes tab and work with the following options:

    Click New on the toolbar to create a work class name. Use the Enter filter text text box to filter a large list of names to locate the one you want to work with.

    Name

    Specifies the name of the work class.

    Type

    Specifies the type of database activity. Select from the drop-down list; valid values are All, Read, Write, Call, DML, DDL, and Load.

    Generate

    Specifies whether or not to generate DDL for this object during Forward Engineering.

  3. Click the General tab to work with work class properties. Properties are dynamic, based on the work class type you have selected.
    Range Type

    Specifies the type of information that you want to specify in for the Range From Value and Range To Value fields. Select from the drop-down list; valid values are Timeron Cost and Cardinality.

    Range From Value (text)

    Specifies the upper limit of the range of either timeron value or cardinality within which the database activity falls, to be part of this work class.

    Range To Value (text)

    Specifies the lower limit of the range of either timeron value or cardinality within which the database activity falls, to be part of this work class.

    Schema

    Specifies the schema name of the procedure the CALL statement will be calling.

  4. (Optional) Click the Where Used tab to view where the object is used within the model.
  5. (Optional) Click the Notestab to view and edit user notes.
  6. (Optional) Click the Extended Notes tab to view or edit user notes.
  7. Click Close.

    The Db2 for LUW Work Class Editor closes.
