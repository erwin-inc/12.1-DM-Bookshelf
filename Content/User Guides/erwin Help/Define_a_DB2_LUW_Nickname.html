Define a Db2 LUW Nickname

Use the Db2 for LUW Table Editor to create a nickname, or work with the properties of an existing Db2 LUW nickname.

To define a Db2 LUW nickname

  1. Right-click a Table in the Model Explorer and select Properties.

    The Db2 for LUW Table Editor opens.

  2. Select a table and work with the following options:

    Click New on the toolbar to create a table. Use the Enter filter text box to filter a large list of names to locate the one you want to work with.

    Physical Name

    Specifies the physical name for the table.

    Schema

    Specifies the schema associated with the selected table. Create or select a schema from the drop-down list.

    Db2/LUW Type

    Specifies the table type. Select Nickname from the drop-down list. The Organize By, Tablespace, Distribution, Partitions, and Volumetrics tabs are disabled based on this value.

    Use Replace Syntax

    Specifies whether or not to use the REPLACE command instead of the CREATE command during Forward Engineering.

    This field applies to Db2 for LUW version 9.7 only.

    Physical Only

    In a Logical/Physical model, specifies that the table does not appear on the logical side as an entity.

    Generate

    Specifies whether or not to generate DDL for this object during Forward Engineering.

  3. Click the Dimensional tab and work with the following options:
    Dimensional Notation Rule Type

    Specifies the role assigned to the table for dimensional modeling. Select from the drop-down list; valid values are Fact, Dimension, and Outrigger.

    Dimensional Notation Update Type

    Specifies the type of update you want to use to update dimensional notations. Select from the drop-down list; valid values are Fixed, Type 1, Type 2, and Type 3.

  4. Click the Data Movement Rule tab and work with the following options:
    Refresh Rule
    Append Rule
    Backup Rule
    Recovery Rule
    Archive Rule
    Purge Rule
  5. Click the General tab and work with the following options:
    Nickname Type

    Specifies the type of the nickname. Select from the drop-down list; valid values are Relational and Non-relational.

    Remote Table Schema

    Specifies the schema to which the selected nickname belongs.

    Remote Table Name

    Specifies the table for which the nickname is created.

    Server

    Specifies the data source (or the server) that contains the table for which the nickname is created.

    Audit Policy

    Specifies the audit policy for the selected nickname. Create or select from the drop-down list.

  6. Click the Table Options tab and work with the following options:
    Name

    Specifies any additional options on the columns of the nickname.

    Value

    Specifies the value for the selected additional option.

  7. Click the Alias tab and work with the following options:
    Name

    Specifies the alias for the nickname.

    Schema

    Specifies the schema for the alias. Create or select from the drop-down list.

  8. Work with the other tabs in the editor to access additional design features, for example:
    • Click the Permission tab and grant permissions to users or roles on the selected table.
    • Click the Style tab and specify a theme for the selected table.
    • Click the Icon tab and select an icon for the selected table.
    • Click the Object Creation Order tab and change the order in which objects of the selected table are created.
  9. (Optional) Click the Commenttab and enter any comments that you want to associate with the object.
  10. (Optional) Click the Where Used tab to view where the object is used within the model.
  11. (Optional) Click the UDPtab to work with user-defined properties for the object.
  12. (Optional) Click the Notestab to view and edit user notes.
  13. (Optional) Click the Extended Notes tab to view or edit user notes.
  14. Click Close.

    The Db2 for LUW Table Editor closes.
