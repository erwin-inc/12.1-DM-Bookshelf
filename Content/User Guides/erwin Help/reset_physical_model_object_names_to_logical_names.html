Reset Physical Model Object Names to Logical Names

You can use the Globally Reset DBMS Property dialog to select the model objects for which you want to reset the physical model object names to logical names.

To reset physical model object names to logical names

  1. Click Model Naming Options on the Tools menu.

    The Model Naming Options dialog opens.

  2. In the Name Mapping tab, click Reset Names.

    The Globally Reset DBMS Property dialog opens.

  3. Choose any of the following options:
    All Table Names

    Resets the physical table names to the logical entity names.

    All Relationship Names

    Resets the physical foreign key constraint names to the logical verb phrase.

    All Index Names

    Resets the indexes to the default name created by the %IndexName macro.

    Selected Column Properties

    Resets the column names to the default name or logical equivalent for the columns selected in the Column Properties list.

    Only Columns that are attached to user-defined Domains

    This option appears when you select Selected Column Properties, it resets the column names only for columns that are attached to a user-defined domain.

    Select Column Properties to Reset

    This option appears when you select Selected Column Properties. Select the properties you want to reset.

    Click OK.

    The dialog closes.

  4. Click OK.

    The dialog closes and the physical model object names are reset to logical names.
