The Notes Editor

Use the Notes Editor to add or update a note. You can add notes for both physical and logical model objects.

Follow these steps:

  1. Open the property editor for an object, go to the Notes tab, and click Edit.

    The Notes Editor dialog appears. The following fields are not self-explanatory:

    Transfer

    Marks the note for transfer to the Extended Notes tab. To transfer a note, click the Transfer between notes and extended notes icon in the toolbar. When you transfer a note from the Notes tab to the Extended Notes tab, the following changes occur:

    • The values in the Created By and Modified By fields are deleted and the current user is displayed in the Author field.
    • The value in the Event field is deleted.
    Status

    Displays the property inheritance icon for the notes that are inherited from the parent table. This column is available only in the Column Editor dialog.

    Comment

    Displays the comment that is entered in the Notes tab. This column also displays the note that is transferred from the Extended Notes tab.

    Migrate

    Migrates the note from a parent table to the child table. For example, in a table named User, you have added a note to the column User_ID and selected the Migrate check box. When you use User_ID as a foreign key in another table, the note is displayed. This column is available only in the Column Editor and Domain Editor dialogs.

  2. Click New to add a new note.
  3. Use the remaining toolbar icons to sort, delete, and move the notes up or down.
