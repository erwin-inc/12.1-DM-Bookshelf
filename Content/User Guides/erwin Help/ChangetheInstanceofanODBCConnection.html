Change the Instance of an ODBC Connection

If you have multiple copies of erwin Data Modeler (DM) running, the ODBC driver can only serve data from one instance. By default, this is the first one started. If you want to change which instance of erwin DM is handling the ODBC connection, you need to first stop the driver on the one that is running and then start the driver on the second copy of erwin DM from the same place.

To change the instance of an ODBC connection

  1. Go to the copy of erwin DM that is running for which you want to stop the ODBC driver and click Options on the Tools tab.

    The Options dialog opens.

  2. Go to the Reporting tab and click Stop ODBC driver for this instance.

    This button's display changes to Start ODBC driver for this instance once the ODBC driver is stopped for the current instance.

  3. Click OK.

    The Options dialog closes.

  4. Go to the other copy of erwin DM that is running for which you want to use the ODBC driver and click Options on the Tools tab.

    The Options dialog opens.

  5. Go to the Reporting tab and click Start ODBC driver for this instance.

    This button's display changes to Stop ODBC driver for this instance once the ODBC driver is started for the current instance.

  6. Click OK.

    The Options dialog closes and you are ready to start constructing a query.
