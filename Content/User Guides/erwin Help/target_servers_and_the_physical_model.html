Target Servers and the Physical Model

Before you define the physical properties of your DBMS, you must select the target DBMS for the physical schema. Use the Target Server dialog to select the target SQL or desktop DBMS for the physical schema, and to define default settings for generating the schema.

Choose a target DBMS before you work on the schema. If you do, DBMS-specific datatypes and name length warnings are provided by default.

When you need to compare the information stored on the server with the information defined in your local model, and import or export data from one to the other, you must create an active connection to the target database. You can connect to the system catalog in the target database using one of these methods:

  • The native interface utilizing vendor-specific functions for connecting to that catalog
  • Open Database Connectivity (ODBC) middleware.
  • For SQL Server and Oracle, you can use Windows Authentication to establish your connection to the database. Use this method if you want the database server to use the current Windows user identification for the connection. To use this method, select the Windows Authentication option in the SQL Server or Oracle Connection dialog.

See the table in the DBMS Connection Software for a list of client software required to connect to each target DBMS that is supported by this product.
