Save a Model in XML Format (XML Export)

You can use XML format to export model data to applications that support XML. XML format replaces the text-based ERX format used in earlier versions of this product. XML format provides structured markup of model metadata according to elements that are defined in the XML Schema.

You can export a model in XML format which is suitable for importing back into erwin Data Modeler.

To export a model in XML format

  1. Select Save As from the File menu.

    The Save As dialog opens.

  2. Enter a file name in the File name box, select one of the following XML format options as the file type from the Save as type dropdown:
    XML Standard Files

    Standard XML format does not expand macros or names. This file is suitable for importing back into erwin Data Modeler.

    XML Standard with Min Info Files

    Only saves the minimum amount of information needed by erwin Data Modeler to reopen the model. Properties that are derived from other properties or that are read-only are not saved. You can choose to not use this option and still use the standard XML format.

    XML Repository Format Files

    The Repository format expands names, macros, and physical names. Derived properties are exported. A file saved in this format cannot be opened with erwin Data Modeler.

  3. Click the Save button.

    The Save As dialog opens.

    You can see the progress of the XML Export. You can cancel the XML Export at any time, and the XML file will not be saved.

    Click the Cancel button to cancel the XML Export.
