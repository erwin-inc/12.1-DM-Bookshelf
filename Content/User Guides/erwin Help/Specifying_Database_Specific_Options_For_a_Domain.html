Specifying Database-Specific Options For a Domain

In a physical model, the <database> tab in the Domain Editor contains options for defining common domain properties such as physical data type and null option and are not database-specific.

However, more options are provided for some of the target databases that require you to define additional properties. The following target databases have additional, database-specific options for defining domain physical properties:

Db2 for i

Db2 LUW

Db2 z/OS

Hive

Informix

Netezza

Oracle

PostgreSQL

Progress

SAS

SQL Server

SAP IQ

Teradata
