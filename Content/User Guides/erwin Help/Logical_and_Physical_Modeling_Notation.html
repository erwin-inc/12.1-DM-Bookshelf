Logical and Physical Modeling Notation

For entity relation data models, you can specify whether to use IDEF1X or IE notation for the objects in the logical and physical model. If you prefer, you can use a different notation to help you distinguish between the different model types.

For dimensional models, you can specify IDEF1X or IE notation in the logical model and select DM (Dimensional Modeling) for the physical model. You can also choose whether to display conformance warnings based on dimensional modeling rules.

For information about applying IE notation, see the article How to Create a Model and follow the steps in the Set Model Properties topic.
