The Notes Tab

The Notes tab lets you add, edit, view, and delete notes to objects. You can add notes apart from adding definitions in the Comment tab and adding comments in the History tab.

The following fields are not self-explanatory:

Status

Specifies the inheritance status of the selected note—overridden or inherited. This option is available only in the Column Editor dialog.

Migrate

Migrates the note from the parent column if this check box is selected in the parent column. For example, suppose in a table named User, you have added a note to the column User_ID and you have selected the Migrate check box. When you use User_ID as a foreign key in another table, the note is displayed. This option is available only in the Column Editor and Domain Editor dialogs.

From release 9.6 onward a new tab, Extended Notes is available for all model objects. You can transfer the notes from the Notes tab to the Extended Notes tab. You can also transfer a note from the Extended Notes tab to the Notes tab.

When you transfer a note from the Extended Notes tab to the Notes tab, the following changes occur:

  • The Name, Importance, and Status information is deleted.
  • The value in the Author field is deleted and the current user is displayed in the Created By field.

The value in the Event field is deleted and User Note is displayed.
