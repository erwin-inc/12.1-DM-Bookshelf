The Model Properties Dialog

The Model Editor includes the following tabs for setting default properties and documenting details for your model:

General

Lets you enter the author and title for the model and review the target server and version. If the model is reverse engineered, the database the model was imported from displays. Select either the Integration DEFinition for Information Modeling (IDEF1X), Information Engineering (IE), or Data Warehousing notation preferences for the model, and whether to use Dimensional, Data Vault, or Data Movement modeling features.

Defaults

Lets you set default null and datatype options and specify default themes. Specify default owner names, and globally reset owner names using the Defaults tab.

Use Data Type in Unification

Specifies whether or not you want the default logical and physical datatypes to be used in unification of two or more tables or entities.

RI Defaults

Lets you set default referential integrity rules for relationships in the model.

History Options

Lets you configure the objects and events for which history is tracked for the model.

Comments

Lets you enter comments for the model.

UDP

Lets you specify a user-defined property value for the model.

Notes

Lets you view the accumulated historical data about your model.
