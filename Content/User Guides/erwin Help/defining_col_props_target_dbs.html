Defining Table Column Properties for Target Databases

In a physical model, the <database> tab in the Table Column Editor contains options for defining common table column properties such as physical data type and null option and are not database-specific.

However, more options are provided for some of the target databases that require you to define additional properties. The following target databases have additional, database-specific options for defining table column properties:

Db2 for i

Db2 LUW

Db2 zOS

Informix

Hive

MariaDB

Oracle

PostgreSQL

Progress
