Copy Objects to a Different Model

You can copy entities, tables, views, or drawing objects to a different model. You cannot copy attributes or columns to a different model.

Copying entities and other objects from a relational to a non-relational database is restricted in erwin Data Modeler (DM).

To copy objects to a different model

  1. Open a diagram and select the objects that you want to copy.
  2. Click Edit, Copy.
  3. Perform one of the following:
    • Create a model
    • Open an existing model
  4. Click Edit, Paste.

    The objects are pasted as new objects in the target model. The new objects are included in the diagram in the same layout and location as in the source diagram. The new objects are displayed in the Model Explorer.

  5. Drag the pasted objects to a blank area of the diagram window if the objects overlap.

If you select individual entities in a subtype relationship and you want to copy the relationship between them, you must select all the entities plus the subtype symbol for the relationship lines to copy correctly. All other relationship lines are copied automatically when both the parent and child entity in the relationship are included in the copy selection set.
