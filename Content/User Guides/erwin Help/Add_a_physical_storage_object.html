Add a Physical Storage Object

You can define physical storage objects, such as a database, tablespace, and segment, for some target servers, and specify parameters for the storage objects. The physical storage objects and parameters you can define are different, based on the target server you select. The physical storage options that you see after you select a target server are customized so you only see the available options for your target server. This customization assists you in managing the physical allocation of objects in your database.

To add a physical storage object

  1. Click Target - <database> on the Model menu and then select an option for the physical object that you want to add (such as Tablespace, Rollback Segment, Database).

    The property editor for the physical object opens.

  2. Click New on the Navigation Grid toolbar.

    A new physical object is added to the Navigation Grid using a default name.

  3. Enter a name in the Name cell.

    See your SQL vendor's documentation for restrictions on naming physical storage objects.

  4. Click the General tab and enter a value for each parameter you want to define for the new object.

    When you add a physical object, any parameters you leave blank are assigned a default value by the target server. For more information about setting physical storage parameters, see your SQL vendor's documentation.

  5. Work with the other tabs to further define storage parameters.
  6. Click Close.

    The physical storage object is added and the property editor closes.

If your target server is Db2 z/OS, Db2 LUW, Hive, Oracle, SQL Server, or Teradata, you can forward engineer the storage objects when you generate the database schema.
