Define Partitions for Oracle Tables, Materialized Views, and Materialized View Logs

To define partitions for Oracle tables, materialized views, and materialized view logs

Do the following:

  • Table - In the Model Explorer, under Tables, right-click Partitions and click New. Or, in the Diagram Area, right-click a table and then click Partition Properties.
  • Materialized View - In the Model Explorer, under Materialized Views, right-click Partitions and click New. Or, in the Diagram Area, right-click a materialized view and then click Partition Properties.

The options available depend on the partitioning option you select. For information on the partitioning option that you require, click the appropriate partitioning option below:

  • Range
  • Hash
  • List
  • Composite-Hash
  • Composite-List

For a materialized view log partition:

  1. In the Model Explorer or in the Diagram Area, right-click a table and select Table Properties. The table editor opens.
  2. Click the Materialized View Log tab.
  3. Select the Materialized View Log check box and then click Edit Properties. The materialized view log editor opens.
  4. Click Partitions to open the partition editor.

For each of the objects, do the following using the partition editor:

  1. (Optional) Click the Subpartitions tab and define the subpartitions for the selected partitions.
  2. (Optional) Click the Tablespace tab and add tablespaces to the selected partitions.
  3. (Optional) Click the UDP tab to work with user-defined properties for the object.
  4. (Optional) Click the Notestab to view and edit user notes.
  5. (Optional) Click the Extended Notes tab to view or edit user notes.
  6. Click Close, the editor closes.
