Create Teradata Replication Rule Sets

In Teradata 13, you can use the Replication Rule Set editor to describe the rules for automatically assigning objects to a replication group when one is created.

To create a Teradata replication rule set

  1. Click Target-Teradata, Replication Rule Sets on the Model menu.

    The Teradata Replication Rule Set Editor opens.

  2. Select a replication group and work with the following options:

    Click New on the toolbar to create a new replication rule set. Use the Enter filter text text box to filter a very large list of replication rule sets to quickly locate the one you want to work with.

    Name

    Specifies the name of the replication rule set.

    Is Default

    Specifies that this replication rule set defines default rules for the replication group.

    Replication Group

    Specifies the replication group that this replication rule set applies to.

    Use Replace Syntax

    Lets you select whether Forward Engineering uses the REPLACE command instead of the CREATE command.

    Generate

    Generates DDL for this object during forward engineering.

  3. On the Ruletab, select the replication rules you want associated with the replication rule set. Existing replication rules display in the tab. Click New to create a new replication rule, or click the Replication Rule icon to open the Replication Rule Editor, for additional options.
  4. (Optional) Click the Comment tab and enter any comments that you want to associate with the object.
  5. (Optional) Click the UDP tab to work with user-defined properties for the object.
  6. (Optional) Click the Notes tab to view and edit user notes.
  7. Click Close.

    The Teradata Replication Rule Set Editor closes.
