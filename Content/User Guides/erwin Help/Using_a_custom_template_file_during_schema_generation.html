Using a Custom Template File During Schema Generation

Forward engineering template files were present in erwin® Data Modeler r7 through r7.2. However, as noted in the documentation of those releases, these template files used an interim format and changes to them were unsupported. Beginning with the r7.3 format, changes to template files are supported. The file format and template syntax will also be supported in future releases of erwin® Data Modeler, and changes you make to the template files will be usable in those future releases. However, as future releases become available, support for a new database version may be added or removed, and the default template files may change. You should update your customized template files with the changes made to the default template files. Information provided with the service pack or release includes information about changes to the default template files. You can also refresh the template files for your current install environment by copying read-only versions of the template files located in the \Program Files (x86)\erwin\Data Modeler r9\BackupFiles\Templates directory.

Prior versions of template files cannot be used in erwin® Data Modeler r7.3 or later. Both the file format and the template syntax have changed.

To use a customized template in the Schema Generation dialog for either Forward Engineering/Schema Generation or Alter Script/Schema Generation, you can browse for and select the appropriate customized template file for your schema generation tasks. If you decide to use the default forward engineering template instead, you can use the Reset Template option.

Important! If you use a custom template file, the file path is saved with the model. If you move the custom template file to another location, the link to the custom template is broken, and the option to use the default template is restored.
