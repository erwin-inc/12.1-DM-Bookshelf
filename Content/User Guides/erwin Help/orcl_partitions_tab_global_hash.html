Oracle Partitions Tab - Global-Hash

The following describes the options available when you select the Global-Hash partitioning type in the Partitions tab:

Available Columns

Displays the columns in the current model. To add a column to the Selected Columns list, click on the column that you want from the Available Columns list, and click the button to add.

Selected Columns

Displays the columns you selected and moved from the Available Columns list. To remove a column from this list and place it back into the Available Columns list, click on the column that you want from the Selected Columns list, and click the button to remove. Use the button and the button to move a column up or down in the list.

Partition definitions

When you select this option, you must select one of the following options for Partition definitions:

Partition by quantity

If you select this option, enter a value in the Number of Partitions list box, and add a tablespace to the Tablespace list box. Click the button to open the Oracle Select Tablespaces dialog.

Partition by name

Select this option to create individual partitions. Enter a value in the Partition Name column. When you click on the button within the Edit Properties column, it opens the Oracle Partition Storage Dialog. When you click on a specific partition name, you can use the button and the button to move it up or down in the list, or click the Delete button to delete it.

The following describes general information regarding the Global-Hash type:

  • Overflow tablespaces are not available for index partitions.
  • LOB storage is not available for index partitions.
