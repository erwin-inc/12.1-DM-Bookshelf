Convert Datatypes When Changing the Target DBMS

When you change the target DBMS, you can convert the datatype assigned to each attribute to the closest datatype available on the new target server. Specific datatype mapping rules are used when converting datatypes.

To convert datatypes when changing the target DBMS

  1. Click Target Database on the Actions menu in a physical model.

    The erwin Data Modeler - Target Server dialog opens.

  2. Select the target DBMS for the current model in the Target SQL DBMS section.
  3. Select the target database version from the <Database> Version list and click OK.

    If the new database supports user-defined datatypes (and the current database does not), a warning message displays about converting your datatypes.

  4. Select one of the following options in the warning message dialog:
    • Click Yes to convert your domains to user-defined datatypes. All overridden datatypes will be reset to restore domain inheritance.
    • Click No to not convert the datatype settings for domains. The datatypes are maintained from the original DBMS.

If you do not automatically convert the domain datatypes, you must manually convert them, or convert to a third DBMS, then convert back to your intended DBMS.
