Define a Db2 LUW Work Action

You use the Db2 for LUW Work Action Editor to set properties for a work action. You associate a work action with a work action set and apply the work action to a work action class. Different properties are displayed on the General tab, based on the action type defined for the work action.

To define a Db2 LUW work action

  1. Click the General tab in the Db2 for LUW Work Action Set Editor.

    Your work actions are listed in the lower part of the tab. You can create a work action, or click the Work Action Editor icon to open the Db2 for LUW Work Action Editor.

    The following options are available when you invoke the Db2 for LUW Work Action Editor:

    Name

    Specifies the name of the work action.

    Work Class

    Specifies the associated work class. Select a work class from the drop-down list.

    Action Type

    Specifies the work action type. Select an action type from the drop-down list. The values in the drop-down list depend on the action set type you have selected in the Db2 for LUW Action Set Editor.

    Enabled

    Specifies that the work action is enabled.

  2. Click the General tab to work with properties specific to the action type specified.
    Service Subclass

    Specifies the service subclass that you want to associate with the selected action type.

    Threshold Predicate Type

    Specifies the threshold predicate type that you want to associate with the selected action type.

    Threshold Value

    Specifies the threshold value that you want to associate with the selected action type.

    Threshold Queued Activity Value Type

    Specifies the threshold queued activity value type that you want to associate with the selected action type. This field is displayed only if you select Concurrent DB Coordinated Activities for Threshold Predicate Type.

    Check Threshold Interval

    Specifies the threshold interval that you want to associate with the selected action type.

    Threshold Collect Activity Data Type

    Specifies the service subclass that you want to associate with the selected action type.

    Threshold Action Execution Type

    Specifies the type of action that is to be taken when the threshold value exceeds. Select from the drop-down list; valid values are Stop and Continue.

    Collect Activity Data For Partition Type

    Specifies how activity data is to be collected for partitions. Select from the drop-down list; valid values are Coordinator Partition and All Partitions.

    Collect Aggregate Activity Data Type

    Specifies to capture aggregate activity data for activities associated with the work class for which the current work action is defined.

  3. Click the Histogram Template tab to define histogram-related properties.
  4. (Optional) Click the Comment tab and enter any comments that you want to associate with the object.
  5. (Optional) Click the UDP tab to work with user-defined properties for the object.
  6. (Optional) Click the Notes tab to view and edit user notes.
  7. Click Close.

    The Db2 for LUW Work Action Editor closes.
