Define a Db2 z/OS File

You can use the File editor to define the physical location of a file that is used in a database for Db2 physical models.

To define a Db2 file

  1. Right-click a File in the Model Explorer and select Properties.

    The File Editor opens.

  2. Select a file and work with the following options:

    Click New on the toolbar to create a file. Use the Enter filter text text box to filter a very large lista large listkly locate the one you want to work with.

    Name

    Specifies the name of the file.

    Schema

    Specifies the default schema for the file. Select a schema from the drop-down list.

    Type

    Specifies the type of file. Select from the drop-down list; valid value is Jar.

  3. (Optional) Click the Permissiontab to work with file permissions.
  4. (Optional) Click the Comment tab and enter any comments that you want to associate with the object.
  5. (Optional) Click the UDP tab to work with user-defined properties for the object.
  6. (Optional) Click the Notes tab to view and edit user notes.
  7. Click Close.

    The File Editor closes.
