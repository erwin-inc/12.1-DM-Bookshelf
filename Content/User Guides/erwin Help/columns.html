Columns

When you add and maintain columns in a table, you must assign physical properties to the column including data type, null option, defaults, validation rules, volumetrics, comments, user-defined properties, display properties, and even storage properties for some target servers. You can manage all column properties in the Table Column Editor Column Editor.

In addition to specifying column properties, you can also assign data type, edit masks, and other column properties using the Domain Editor to define domains. A domain is a named set of column properties that you can attach to one or more columns in a model to quickly and accurately define their properties.

You can also review, move, copy, and modify columns using the Model Explorer.

The unique name rules are not enforced for attribute and column names inherited from domains.
