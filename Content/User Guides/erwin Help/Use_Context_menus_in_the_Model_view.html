Use Context Menus in the Model View

There are various context menus within the Model view of the Model Explorer. The default context menu for a collection node provides you with the ability to create a new instance of the contained type.

If semantics require additional information in order to complete the creation of a new object, a popup modal dialog displays to gather that information. For example, creation of a new Relationship requires the specification of a parent entity, a domain collection context menu exists so that you can control the topological display of domains, and the default context menu for an object instance provides you with the ability to either go to the object in the diagram, delete the object, or invoke the property editor for the object. If semantics do not permit the deletion of an object (for example, a primary key object), the delete menu item is inactive.

To use a context menu in the Model Explorer, select a model object and right-click the mouse button.
