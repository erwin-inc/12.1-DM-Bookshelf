Bulk Editor Wizard Option Sets

The Object Types, Property Types, and Object Instances pages in the Bulk Editor Wizard contain functionality that lets you save and reload a stored definition of selected objects and properties. This saves time by allowing you to make your selections once, and then save those selections as an option set within the current model or as an external XML file. You do not need to make selections on every page in order to save or change an option set.

An option set stored within the model preserves all object type, property type, and object selection information. However, an option set saved as an external XML file only saves object type and property type selections, not object-level selections. This is because XML files can be shared across models, and object-level information is specific to the current model only.

You manage option sets by using the following controls on the option set toolbar:

Option Set

Specifies the name of the option set. Use the drop-down control to select and load an existing option set.

Open

Opens a File Open dialog so you can select and open an option set from an external XML file.

Save

Updates the current option set. If it is part of the current model, it is updated there. If it was opened from an external XML file, that file is updated.

New

Opens the Save Current Option Set dialog so you can create a new named option set and store it in either the current model or as an external XML file.

Delete

Deletes the option set shown in the Option Set name drop-down area.
