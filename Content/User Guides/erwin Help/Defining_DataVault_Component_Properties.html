Defining Data Vault Component Properties

To edit a Data Vault component's properties, follow these steps:

  1. In the Model Explorer, right-click the component and click Properties.
    The corresponding object editor appears. For example, SQL Server Data Vault 2.0 Hub Editor.

  2. Configure the components properties. Refer to the following table for property descriptions.

    Tab

    Section

    Property

    		 Description

    Additional Information
        Name Specifies the Data Vault component's name  

    Table Class Description

    		 Specifies the Data Vault component's type  
    Prefix Specifies the prefix to be attached to the table name to indicate its Data Vault 2.0 Component type For example, for a Hub, the prefix is H_
    Suffix Specifies the suffix to be attached to the table name to indicate its Data Vault 2.0 Component type For example, for a Hub, the suffix is _HUB
    Attributes Component Color Options

    Color

    Specifies the color to be used for a table on the model diagram when its Data Vault 2.0 Component is set up

    		  
    Component Attributes Name Specifies the attribute name  
    Domain Parent Specifies the attribute's domain parent  
    Physical Data Type Specifies the attribute's physical data type  

    Column Class Description

    		 Specifies the attribute's class  
    Mandatory/Optional Specifies whether the attribute is mandatory or optional  
    Append Table Name as Prefix/Suffix Specifies whether the table name should be appended as a prefix or suffix to the Data Vault component
