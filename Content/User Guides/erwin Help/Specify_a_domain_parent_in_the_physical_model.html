Specify a Domain Parent in the Physical Model

You can work with features in the Domain Editor to specify a domain parent in a physical model. Once a parent domain is selected, the inheritable properties of the parent domain are automatically inherited by the domain.

You cannot change the domain parent for any of the default domains (<default>, Blob, Datetime, Number, and String).

To specify a domain parent in the physical model

  1. Click Domains on the Model menu.

    The Domain Editor opens.

  2. Click the name of the domain in the Navigation Grid for which you want to specify a domain parent.

    To filter a large list of domains, enter the name (or the first few characters) of the domain for which you want to specify a domain parent in the Enter filter text box.

  3. Click the General tab and work with the following option:
    Domain Parent

    Lists the domain parent hierarchy and lets you specify the parent domain for the selected domain in the Navigation Grid. Select the check box of the domain that you want to be the parent domain.

  4. Click Close.

    The domain parent is specified and the Domain Editor closes.
