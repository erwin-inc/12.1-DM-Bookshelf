Using Model Templates - Best Practices

Review the following information to maximize the benefits of model templates:

  • Create a model template using the same target server as the type of model you want to create. Model templates can work with models using a different target server, but the conversion process to make the model objects match is not always predictable. Some model objects are specific to target servers, such as SQL Server Cryptographic Providers, or Teradata Replication Groups.
  • If you want to have settings such as Unique Naming applied, specify it in the template model, not merely the target model. The settings in the template model override the settings in the target model. The alternative is to use the Ignore synchronization of the root Model object setting in the Model Template Synchronization Wizard. Decide whether this alternative is acceptable, because this setting blocks synchronization of all properties on the model.
