SQL Server Property Editors

The following list summarizes support for SQL Server physical property editors. These editors provide additional support beyond the basic erwin® Data Modeler physical objects such as tables, indexes, scripts, and stored procedures, and are used to define more specific SQL Server property values.

  • Aggregates
  • Always Encrypted Keys
  • Application Roles
  • Assemblies
  • Asymmetric Keys
  • Certificates
  • Credentials
  • Cryptographic Providers
  • Database Audit Specification
  • Database Encryption Keys
  • Database Roles
  • Databases
    • File Group
    • Triggers
  • Files
  • Full-Text Catalogs
  • Full-Text Index
  • Full-Text Stoplists
  • Functions
  • Logins
  • Partition Functions
  • Partition Schemes
  • Physical Files
  • Permissions
  • Resource Pools
  • Schemas
  • Sequences
  • Server Audit
  • Server Audit Specification
  • Stored Procedures
  • Symmetric Keys
  • Synonyms
  • Tables
    • ColumnStore Indexes
    • Spatial Indexes
    • XML Indexes
    • Indexes
    • Triggers
  • Triggers
  • User Ids
  • Views
    • View Index
    • View Triggers
  • Workload Groups
  • XML Schema Collections

For more information about specific SQL Server property values, see your SQL Server database documentation. The erwin® Data Modeler documentation for the physical property editors provides brief descriptions of the controls in each dialog and tab, which you can use as a point of reference while working with database design features.
