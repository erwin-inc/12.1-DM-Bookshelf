Specify Default Trigger Directories for Progress

You can use the Default Trigger Directories/Prefixes dialog to organize erwin Data Modeler-generated Progress triggers. You can create, write, and delete triggers in separate directories, and you can choose to use the dump file name, rather than the table name, as the trigger name.

To specify default trigger directories for progress

  1. Click Choose Database on the Database menu.

    The erwin Data Modeler -- Target Server dialog opens.

  2. Select Progress as the target DBMS in the physical model, and click Trigger Directory.

    The Default Trigger Directories/Prefixes dialog opens.

  3. Select any of the following options in the Default Trigger Directories/Prefixes dialog:
    • To specify the relative directory path to the directory in which you want to store the default create triggers that are generated, enter the path in the Create Triggers box.
    • To specify the relative directory path to the directory in which you want to store the default write triggers that are generated, enter the path in the Write Triggers box.
    • To specify the relative directory path to the directory in which you want to store the default delete triggers that are generated, enter the path in the Delete Triggers box.
    • To use the dump file name, if it exists, rather than the table name, as the trigger name, select the Use Dump File Name As Trigger Name check box.
  4. Click OK.

    The Default Trigger Directories/Prefixes dialog closes.

  5. Click OK.

    The erwin Data Modeler -- Target Server dialog closes.
