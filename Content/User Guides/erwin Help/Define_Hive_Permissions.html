Define Hive Permissions

Use the Hive Permission Editor to define a permission for a database, table, view in a Hive physical model.

By default, the admin role is not in the current roles. While generating a table or view permission, on the Forward Engineer Schema Generation dialog box, under Other Options, ensure that you select Set Admin Role.

To define a Hive permission

  1. In the Model Explorer, under Databases, Tables, or View, right-click Permissions and click New.

    An instance of Permission is created.

  2. Right-click the instance and click Properties.

    The Hive Permission Editor opens.

  3. In the navigation grid, select the permission that you want to define and work with the following options:

    Click New on the toolbar to create a permission. Use the Enter filter text box to filter a very large list of always encrypted keys to quickly locate the one that you want to define.

    Name

    Specifies the name of the permission. Set the name of the permission in this field.

    With Grant

    Specifies whether the user can grant this permission to other users or not.

  4. On the General tab, work with the following options:
    Permissions to Grant

    Specifies the permissions that are granted to the selected permission. Select or clear check boxes for specific permissions, or select the All Permissions check box to grant all permissions to the permission.

    Principals to Grant

    Specifies the principals that are granted to the selected permission. Select or clear check boxes for specific principals.

    Click an icon to open one of the security editors (User ID user id editor icon , Application Role application role editor icon , or Database Role database role editor icon ) to further define a principal.

  5. (Optional) Click the UDP tab to work with user-defined properties for the object.
  6. (Optional) Click the Notestab to view and edit user notes.
  7. (Optional) Click the Extended Notes tab to view or edit user notes.
  8. Click Close.

    The permission is defined and the Hive Permission Editor closes.

For more information, refer to Hive documentation.
