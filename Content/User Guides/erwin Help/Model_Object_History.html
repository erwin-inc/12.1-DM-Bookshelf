Model Object History

When you create or change a model object, you can select to trace the history of the change. You can save historical information for the entities, attributes, tables, columns, and views of your model, and for the model itself.

In the previous versions of the application, you saved historical information for the following events:

  • You create a logical model and add entities and attributes.
  • You create a physical model and add tables and columns.
  • You derive a physical model from a logical model and create tables and columns in the derived physical model.
  • You split a logical/physical model into independent logical and physical components.
  • You apply a transformation to two tables and create a single table.
  • You reverse engineer a database or script file and create tables and columns in the new model.
  • You add a model source that contributes tables and columns to a model.
  • You migrated a foreign key from one table to another.
  • You upgraded from a previous version of the data modeling tool.
  • You imported an object from a database, XML file, or database script.
  • You imported an object through API.

Now, apart from the existing events, you can save historical information for the following events as well:

  • You copy an entity or attribute.
  • You move an attribute.
  • You import an entity or attribute from an Active Model Template.
