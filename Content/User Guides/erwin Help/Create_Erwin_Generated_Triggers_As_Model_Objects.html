Create erwin-Generated Triggers As Model Objects

In versions of erwin® Data Modeler prior to r7.3, erwin-generated triggers did not exist as objects in the model, they were created during the forward-engineering process, but were not recognized during reverse-engineering. You can select to treat erwin-generated triggers as model objects in order to include the triggers in alter scripts, Complete Compare scenarios, and as part of model templates.

Use the following process to generate erwin-generated triggers as model objects:

  • Click Model Properties on the Model menu. On the RI Defaults tab of the Model Editor, select the "Automatically Create RI Triggers" check box. When you select this check box, you are prompted with an option to create RI triggers for existing relationships. If you click Yes, the RI triggers are created for existing relationships, and the option is enabled for the automatic creation of triggers for new objects. If you click No to the prompt, RI Triggers are created only for new relationships. The check box is not selected by default.
  • Click Forward Engineer, Schema on the Actions menu. Click the Trigger option and review the choices listed in the Trigger pane. Select the check boxes for the triggers you want to generate during forward engineering. Be sure to select the "Automatically Generate Missing RI Triggers" check box to generate RI triggers for tables/relationships that do not have RI triggers attached.

When you reverse engineer, you can select to include triggers in the reverse engineer process. In the Reverse Engineer - Set Options dialog, select the "Include Generated Triggers" check box to include the erwin-generated triggers.
