Select Generic ODBC

If you have to use generic ODBC for your database connection in a physical model, follow these steps to connect using ODBC.

To select generic ODBC

  1. Click Target Database on the Actions menu.

    The erwin Data Modeler - Target Server dialog opens.

  2. Select the ODBC/Generic option for the Target SQL DBMS, then select the settings for the ODBC/Generic option:
    ODBC/Generic Version

    Lets you specify the ODBC version to use for the connection. Select a version number from the drop-down list.

    Default ODBC/Generic Datatype

    Lets you specify the default datatype for new columns. Select a data type from the drop-down list.

    Default Non-Key Null Option

    Lets you specify the default Null option for new columns. Select or clear the check box.

    A dialog opens asking you if you want a report on unconverted datatypes.

  3. Select one of the following actions:
    Yes

    Specifies to run the Data Type Mapping Exception Report. Save or print the report, then click Close to close the dialog.

    No

    Specifies to not run the Data Type Mapping Exception Report.

    The ODBC Connection dialog opens.

  4. Enter information in the following fields, then click Connect.
    User Name

    Lets you enter the user name for the ODBC connection.

    Password

    Lets you enter the password for the ODBC connection.

    Invoke ODBC Administrator

    Lets you use the connection dialog that the ODBC data source supports. The Select Data Source dialog opens where you must enter the information specific to your environment that is necessary to connect to ODBC.

    The ODBC Connection dialog closes.
